## 🧩 Overview

**Sushi-Train** is a playful but technically robust **full-stack demo app** built with **Spring Boot**, **Angular**, **PostgreSQL**, **Docker**, and optionally **Kafka/Redpanda**.

It simulates a Japanese conveyor-belt sushi restaurant where:

- Plates circulate automatically on a rotating belt 🍽️
- Guests (Seats) can pick plates and place orders
- Operators can add new plates, change belt speed, and monitor activity

The project is both a **fun learning sandbox** and a **portfolio showcase** of modern backend and frontend design, including event-driven architecture and domain-driven design principles.

---

## 🎯 Purpose

The Sushi-Train project is designed to:

- Learn and apply **Spring Boot**, **Angular**, **PostgreSQL**, **Kafka**, **Docker**, and **Kubernetes**.
- Experiment with **Domain-Driven Design** and **Event Streaming** in a playful context.
- Create a well-documented, visually engaging **portfolio project**.
- Deploy locally via `docker compose` and optionally on the cloud (Phase 4).

## ✨ Features (Phase 1 — “Cute Demo”)

- Conveyor-belt simulation with rotating plates
- Multiple **plate tiers** (different prices in Yen)
- **Seats and Orders**: pick plates, view totals, and check out
- **DDD + Ports & Adapters** structure in backend
- **WebSocket live updates** for belt and seat view
- Run everything locally with `docker compose`

## 🧰 Getting Started (Local Development)

1. **Clone the repository**

   ```bash
   git clone https://github.com/lorenzipsum/sushi-train.git
   cd sushi-train
   ```

2. **Start the stack**

   ```bash
   docker compose up --build
   ```

   Optional: copy `.env.example` to `.env` first if you want to override ports, credentials, or runtime settings.
   The frontend proxy upstream can be changed with `FRONTEND_API_UPSTREAM`, which is useful when the backend runs as a separate cloud service later.

3. **Open in your browser**

   - Frontend (Angular): [http://localhost:4200](http://localhost:4200)
   - Backend API: [http://localhost:8088](http://localhost:8088)
   - Swagger (if enabled): [http://localhost:8088/swagger-ui.html](http://localhost:8088/swagger-ui.html)

4. **Try it out**
   - Use the _Operator view_ to add plates and control the belt.
   - Use the _Seat view_ to pick plates and watch your order grow in real time.

## 🏗️ Architecture Highlights

- **Ports & Adapters (Hexagonal)** architecture
- **Domain Events** documented and versioned
- **Clean separation** of domain, application, and infrastructure
- **Rotation Offset** system for efficient belt animation
- **Ready for streaming** — switch WebSocket → Kafka with minimal changes

For details, see [Architecture Overview](https://github.com/lorenzipsum/sushi-train/blob/main/docs/architecture.md).
