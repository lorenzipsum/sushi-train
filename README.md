## ğŸ§© Overview

**Sushi-Train** is a playful but technically robust **full-stack demo app** built with **Spring Boot**, **Angular**, **PostgreSQL**, **Docker**, and optionally **Kafka/Redpanda**.

It simulates a Japanese conveyor-belt sushi restaurant where:

- Plates circulate automatically on a rotating belt ğŸ½ï¸
- Guests (Seats) can pick plates and place orders
- Operators can add new plates, change belt speed, and monitor activity

The project is both a **fun learning sandbox** and a **portfolio showcase** of modern backend and frontend design, including event-driven architecture and domain-driven design principles.

## ğŸ§° Getting Started (Local Development)

1. **Clone the repository**

   ```bash
   git clone https://github.com/lorenzipsum/sushi-train.git
   cd sushi-train
   ```

2. **Start the stack**

   ```bash
   docker-compose up --build
   ```

3. **Open in your browser**

   - Frontend (Angular): [http://localhost:4200](http://localhost:4200)
   - Backend API: [http://localhost:8088](http://localhost:8088)
   - Swagger (if enabled): [http://localhost:8088/swagger-ui.html](http://localhost:8088/swagger-ui.html)

4. **Try it out**
   - Use the _Operator view_ to add plates and control the belt.
   - Use the _Seat view_ to pick plates and watch your order grow in real time.
