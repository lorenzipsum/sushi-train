## Usage

1. Create one file per session: `YYYY-MM-DD-topic.md`.
2. Keep entries concise and decision-oriented.
3. If a conversation creates a long-lived technical decision, add or update an ADR in `docs/adr/`.
4. Update this index when a new session file is added.
